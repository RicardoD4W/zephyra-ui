# z-vscroller

A virtual scroller component for rendering large lists efficiently. It only renders the visible items in the viewport, improving performance for long lists. Open the devtools and inspect the component, you will see that only the visible items are rendered.
And even though it is "loading" with 10,000 elements and images, the DOM does not overload or take long to load

## Properties

| Prop Name   | Type   | Default Value  | Other Values                    |
|-------------|--------|----------------|---------------------------------|
| `slot` | `HTML` | `None` | `<div>Element 1</div><div>Element 2</div><div>Element 3</div> ... <div>Element 1000</div>` |

## Events
`none`

## CSS Parts

You can controll the size of the vs-scroller using simple CSS like that:

```css
  z-vscroller {
    width: 500px;
    height: 500px;
  }
```

> `IMPORTANT!`
> It would not be correct to use variable units such as: '%', 'vh', 'dvh' in the height.



| Part name  | Description                             | 
|--------------|-----------------------------------------|
| `inner` | The main container. Here the items are rendered. | 
| `item` | The individual items within the scroller. | 

---
